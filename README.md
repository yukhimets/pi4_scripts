# 🛡️ Raspberry Pi Transparent Network Sniffer & AP Compliance Kit

Комплекс скриптов для превращения Raspberry Pi в полностью автономный, прозрачный и программно-аппаратный сниффер сетевого трафика (Transparent Bridge Sniffer) с возможностью изолированного беспроводного управления.

Проект разработан для безопасного анализа сетевых протоколов, проведения аудита информационной безопасности и сбора дампов трафика в полевых условиях без демаскировки устройства в исследуемой инфраструктуре.

---

## 🚀 Архитектурные особенности решения

1. **Прозрачный сетевой мост (L2 Bridge):** Физические сетевые интерфейсы объединяются в виртуальный коммутатор `br0`. На них полностью отключается IP-адресация и обработка DHCP-запросов. Устройство работает как невидимый сквозной патч-корд, не генерируя собственного трафика в проверяемой сети.
2. **Изолированный контур управления:** Скрипт развертывает выделенную Wi-Fi точку доступа. За счет жесткого разделения метрик в ядре Linux, весь трафик администрирования (SSH, AnyDesk, VNC) маршрутизируется строго через беспроводной интерфейс, полностью исключая утечку пакетов управления в исследуемый проводной сегмент.
3. **Отказоустойчивость и защита накопителя:** Сбор дампов осуществляется низкоуровневой утилитой `dumpcap` на виртуальном терминале `TTY3`. Система циклически нарезает `.pcap` файлы фиксированного размера строго на внешний USB-накопитель. Если флешка отсутствует или повреждена, скрипт блокирует захват, предотвращая переполнение внутренней SD-карты и падение операционной системы.

## 🛠️ Пошаговая инструкция по развертыванию

Комплекс рассчитан на установку поверх чистой операционной системы **Raspberry Pi OS** (Debian Buster / Bullseye / Bookworm). Перед началом убедитесь, что малинка временно подключена к Интернету для загрузки необходимых пакетов.

### Шаг 1. Клонирование репозитория
Откройте терминал на Raspberry Pi и скачайте проект:
```bash
git clone [https://github.com/yukhimets/pi_scripts.git]
cd pi_scripts
```

### Шаг 2. Развертывание Wi-Fi точки доступа оператора
Запустите скрипт конфигурации изолированной беспроводной сети:
```bash
sudo chmod +x setup_ap.sh && sudo ./setup_ap.sh
```
*По окончании работы скрипта малинка поднимет собственную сеть. Переключите ваш рабочий ноутбук/планшет на эту точку доступа:*
* **SSID:** `RPI4_AP`
* **Password:** `11111111`
* **IP-адрес малинки для связи:** `192.168.1.1`

### Шаг 3. Конфигурация системы и сниффера

⚠️ **ВАЖНО (Проверка сетевых карт):** По умолчанию конфигуратор настроен на встроенную сетевую карту `eth0` и внешнюю USB-сетевуху `eth1`. Если в вашей системе интерфейсы называются иначе (проверить можно командой `ip link show`), откройте файл `setup_run-dumpcap.sh` и в самом верху в блоке **НАСТРОЙКИ ЖЕЛЕЗА** укажите ваши имена интерфейсов:
```bash
IFACE_1="ваша_карта_1"
IFACE_2="ваша_карта_2"
```

Если имена стандартные (`eth0`/`eth1`), просто запустите инсталлятор:
```bash
chmod +x setup_run-dumpcap.sh && ./setup_run-dumpcap.sh
```

**Что инсталлятор сделает автоматически:**
* Обновит репозитории и установит зависимости: `tshark` (`dumpcap`), `bridge-utils`, `conspy`.
* Соберет прозрачный мост `br0` и жестко заблокирует для него сетевой менеджер `dhcpcd`.
* Автоматически скопирует рабочий скрипт `run-dumpcap.sh` на Рабочий стол (`/home/pi/Desktop/`) с выставлением правильных прав доступа.
* Создаст на Рабочем столе символический ярлык `traffic_flash` для быстрого доступа к логам на флешке.
* Интегрирует глобальный системный алиас для быстрой проверки.
* Пропишет задачу автозапуска в `systemd` и отправит устройство в финальную перезагрузку.

## 📊 Памятка оператора (Инструкция к применению)

### 1. Подключение устройства "в поле"
1. Вставьте в USB-порт Raspberry Pi **любую** флешку, предварительно отформатированную в файловую систему **exFAT** или **FAT32**.
2. Подключите кабель от источника сети (например, от роутера или коммутатора провайдера) в интерфейс `eth0`.
3. Подключите кабель от исследуемого оборудования (целевой ПК, сетевой принтер, банкомат и т.д.) в интерфейс `eth1`.
4. Подайте питание на Raspberry Pi. Устройство загрузится в течение 30-40 секунд, прозрачно соединит порты между собой и автоматически начнет захват трафика.

### 2. Живой контроль пакетов в один клик (`traffic`)
Подключитесь к Wi-Fi сети `RPI4_AP`, зайдите на устройство по SSH (или откройте окно терминала внутри AnyDesk/VNC) и введите быструю команду:
```bash
traffic
```
Скрипт мгновенно подключится к виртуальному экрану `TTY3`. Перед вами откроется интерактивный счетчик пакетов `dumpcap` в реальном времени:
* **Индикатор `Packets: X`** — должен динамически увеличиваться. Это подтверждает, что мост пропускает пакеты и они фиксируются.
* **Индикатор `File: ...`** — показывает текущий файл нарезки на флешке.
* **Выход из режима просмотра:** Быстро нажмите клавишу **`Esc` 3 раза**, чтобы вернуться в стандартную консоль. Сниффер при этом продолжит беспрерывно писать логи в бэкграунде.

### 3. Мониторинг файлов через консоль
Убедиться, что файлы создаются физически на накопителе и весят ровно по 5 Мб, можно через созданный ярлык:
```bash
ls -lh /home/pi/Desktop/traffic_flash/
```

### 4. Изменение фильтров перехвата (BPF)
Если вам необходимо отсечь лишний мусор (например, торренты) и собирать только определенный тип данных:
1. Откройте рабочий файл `/home/pi/Desktop/run-dumpcap.sh`.
2. Найдите переменную `FILTER=""` и впишите туда стандартный BPF-фильтр.
   * *Пример для сбора только веб-трафика:* `FILTER="port 80 or port 443"`
   * *Пример для сбора трафика конкретного IP:* `FILTER="host 192.168.1.50"`
3. Сохраните файл и примените настройки перезапуском службы:
   ```bash
   sudo systemctl restart getty@tty3.service
   ```

### 5. Извлечение собранных данных
Когда сессия аудита завершена:
1. Остановите службу захвата, чтобы корректно финализировать pcap-файлы: `sudo systemctl stop getty@tty3.service` (или просто выключите питание малинки).
2. Извлеките USB-накопитель.
3. Вставьте флешку в ваш рабочий ПК. Внутри папки `traffic` вы найдете структурированную хронологическую нарезку файлов `.pcap`, готовую для глубокого анализа в утилите **Wireshark**.
